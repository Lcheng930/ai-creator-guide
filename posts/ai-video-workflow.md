## 为什么需要工作流？

多数人用 AI 做视频的方式是：打开即梦/可灵 → 输入提示词 → 等生成 → 不满意 → 重新生成 → 还是不满意 → 凑合用。

这不是"做视频"，是"抽卡"。

真正的 AI 视频制作是一套**标准化的工业管线**——每个环节有确定的输入输出、有质量检查标准、有自动化脚本。你不再是一个人在抽卡，你是一个工厂的厂长。

> **核心原则**：AI 管生成，人管决策。工作流的意义是把重复劳动交给机器，把创意判断留给自己。

## 第一站：剧本阶段

### 选题分析

先用 AI 做选题调研，而不是拍脑袋：

**Prompt 模板：**

```
你是一个短视频运营专家。分析以下赛道最近30天的爆款内容趋势：

赛道：[AI短剧/电商带货/文旅宣传/知识科普]
平台：[抖音/快手/B站/视频号]

输出：
1. 最近爆款的5个选题方向（附播放量数据）
2. 每个方向的内容公式（开头/冲突/结尾结构）
3. 机会窗口判断（热度上升/饱和/下降）
4. 建议切入的具体选题
```

**工具推荐**：

| 工具 | 用途 | 特点 |
|------|------|------|
| DeepSeek / Claude | 剧本创作、选题分析 | 中文理解最强 |
| 巨量算数 | 抖音热点趋势 | 官方数据 |
| 新榜 / 飞瓜 | 跨平台热点追踪 | 竞品分析 |

### 剧本格式标准

AI 视频剧本需要比传统剧本更精确——因为你要把它喂给下一环节的 AI。推荐使用**三栏剧本格式**：

```
| 时间 | 画面描述（给AI生成工具）        | 旁白/对白                      | 音效/BGM       |
|------|-------------------------------|------------------------------|----------------|
| 0-3s | 赛博朋克街道夜景，霓虹灯闪烁，    | "2067年的东京..."             | 电子氛围音     |
|      | 雨水倒映灯光，镜头缓慢推轨前进    |                              | 低频鼓点       |
| 3-7s | 主角特写：半张脸被霓虹照亮，      | "她知道自己只剩24小时..."      | 紧张弦乐渐强   |
|      | 另外半张隐在阴影中，眼神警觉      |                              |                |
```

**关键规则**：
- 画面描述要**视觉化**，不要抽象（"悲伤的氛围"→"灰蓝色调，雨滴沿窗玻璃下滑"）
- 每个镜头标注**运镜方式**（固定/推轨/横移/跟拍）
- 对白控制在**每镜1-2句**，AI 配音有字数限制

## 第二站：分镜阶段

### 从剧本到提示词

这是工作流最关键的环节。剧本里的"赛博朋克街道夜景"需要翻译成 AI 能精确理解的提示词。

**分镜提示词五层结构：**

```
[主体描述] → [环境场景] → [光线设定] → [技术摄影] → [风格美学]
```

**示例——把剧本一行的"画面描述"展开为完整提示词：**

> 主体：东京赛博朋克街道，霓虹灯招牌密集悬挂，约15-20块不同大小的全息广告牌
> 环境：雨后的湿沥青路面，倒映霓虹灯光，路面积水形成镜面反射。远景有悬浮飞行器尾灯轨迹
> 光线：多光源——霓虹招牌以品红(#ff00ff)和青色(#00ffff)为主，路灯钠灯橙色(#ff8800)，地面反射光为混合色
> 技术：广角24mm，低角度仰拍，大景深全焦，HDR高动态范围
> 风格：赛博朋克2077风格，PBR写实渲染，HDR高动态范围

### 风格帧

在批量生产前，先花 20 分钟做一个**风格帧**——用一段完整的提示词生成一张概念图，确认整体视觉方向。

**风格帧检查清单**：
- [ ] 色调是否符合内容调性（温馨=暖色、悬疑=冷色、科技=蓝紫）
- [ ] 角色设计是否贯穿一致（服饰、发型、体型）
- [ ] 光线方案是否可行（避免过于复杂的光源组合）
- [ ] 整体质感是否达标（写实度、细节密度）

风格帧通过后再进入批量生成。这是**花 20 分钟省 2 小时**的投资。

## 第三站：生成阶段

### 工具矩阵

| 工具 | 最佳场景 | 分辨率 | 速度 | 成本 |
|------|------|:--:|:--:|:--:|
| **Seedance 2.0** | 真人风格短剧、需要首尾帧控制 | 1080p | 中 | 中 |
| **可灵 (Kling)** | 创意特效、超现实场景 | 1080p | 快 | 低 |
| **即梦 (Jimeng)** | 动漫/二次元、轻量快速 | 720p-1080p | 极快 | 极低 |
| **Runway Gen-4** | 电影级画质、精细控制 | 4K | 慢 | 高 |
| **Pika 2.0** | 社交媒体短视频、趣味效果 | 1080p | 快 | 低 |

### 批量生成策略

```bash
# 批量生成脚本框架
# 输入：分镜CSV（镜头编号, 提示词, 时长, 运镜方式）
# 输出：对应视频片段

for each 镜头 in 分镜表:
    1. 用主提示词生成 3 个候选版本（同提示词×3）
    2. 快速筛选：去模糊、去变形、去不自然的运动
    3. 最佳版本 → 入库，标记镜头编号
    4. 如果3版都不满意 → 调整提示词 → 再生成2版
```

> **质量铁律**：每个镜头至少生成 3 个候选。AI 视频的随机性意味着第一版就满意的概率不到 30%。

### 角色一致性控制

AI 视频最大的痛点是**换镜头就换脸**。解决方案：

| 方法 | 效果 | 适用场景 |
|------|:--:|------|
| **参考图锁定**：每个角色1张正面+1张侧面参考图 | ⭐⭐⭐⭐ | Seedance 2.0 |
| **详细文字描述**：面部特征写到毛孔级（见提示词工程篇） | ⭐⭐⭐ | 所有工具 |
| **LoRA 微调**：训练角色模型 | ⭐⭐⭐⭐⭐ | 长剧集、系列内容 |
| **后期修复**：同一角色多镜头取最佳帧→手动替换 | ⭐⭐ | 短内容应急 |

## 第四站：后期阶段

### 剪辑拼接

AI 生成的素材通常有这些问题：
- 片段间转场生硬
- 部分帧出现形变
- 画面抖动

**剪辑流程：**

```
1. 粗剪：按剧本时间线排列所有片段
2. 去瑕疵：逐片段检查形变帧 → 裁剪或替换
3. 转场：添加交叉溶解（0.3-0.5秒）或硬切
4. 稳定：对抖动片段用剪映/AE的稳定功能
5. 调色：统一所有片段的白平衡和曝光
```

**工具推荐**：剪映专业版（免费、AI 功能丰富）、DaVinci Resolve（专业调色）。

### AI 配音

| 工具 | 音色数量 | 中文质量 | 情感控制 |
|------|:--:|:--:|:--:|
| **剪映** | 100+ | ⭐⭐⭐⭐ | 基础 |
| **微软 Azure TTS** | 300+ | ⭐⭐⭐⭐⭐ | 支持SSML标签调节 |
| **ElevenLabs** | 1000+ | ⭐⭐⭐ | 精细情感参数 |
| **Fish Audio** | 开源 | ⭐⭐⭐ | 可微调 |

**配音规范**：
- 语速：中文 3-4 字/秒（短视频可稍快）
- 停顿：句号 0.5s，段落 1s
- 音色一致性：同一角色全片用同一音色

## 第五站：发布与数据闭环

### 多平台适配

| 平台 | 时长 | 比例 | 封面要点 |
|------|:--:|:--:|------|
| 抖音 | 15-60s | 9:16 | 大字标题+人物表情特写 |
| B站 | 3-15min | 16:9 | 信息密度高的标题卡片 |
| 小红书 | 15-90s | 3:4 | 高饱和度画面+治愈系文字 |
| 视频号 | 30-120s | 9:16 | 正能量+情感共鸣标题 |

### 发布时间窗口

基于 2026 年最新的平台流量数据：
- **抖音**：工作日 12:00-13:00 / 18:00-20:00，周末 10:00-12:00
- **B站**：工作日 12:00-14:00 / 18:00-22:00
- **小红书**：工作日 7:30-8:30 / 12:00-13:00 / 20:00-22:00
- **视频号**：工作日 6:30-8:00 / 20:00-22:00

### 数据复盘模板

每发布 5 条视频后复盘一次：

```
1. 完播率：>30% 合格，>50% 优秀
2. 互动率（点赞+评论+收藏）：>3% 合格，>8% 爆发
3. 粉丝转化：每条带来多少新关注
4. 问题诊断：
   - 前3秒跳出率高 → 开头不够抓人
   - 中途跳出 → 节奏有问题
   - 互动低 → 缺少"钩子"引导互动
```
